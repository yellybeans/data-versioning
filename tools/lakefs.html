LakeFS

Dadurch, dass Data Versioning noch ein relativ neu akutes Problemfeld darstellt, entwickeln sich viele Tools noch weiter, daher kann das folgende Wissen veraltet sein. Falls Ihnen etwas auffällt, wenden sie sich bitte an die Autoren.

LakeFS ist eines der ersten und populärsten Data Versioning Tools, welches sich immernoch stetig weiterentwickelt und verbessert. LakeFS ist dabei eines der git-like Versioning Tools, somit ist es eines der Tools, welches die üblichen Git-Befehle für die Versionskontrolle benutzt.

Es ist dabei sowohl für Data Science Workloads als für das Data Engineering nützlich.

Für einen Data Scientist bringt LakeFS Vorteile, wie z.B.: Robust Data Pre-Processing: Data cleaning, outlier handling, filling in missing values, etc. Ensure your data pipelines for pre-processing are robust and provide high quality. Deduplicated Experimentation: Use lakeFS branches to run experiments in parallel with zero-copy clones in a fully deduplicated data lake, allowing you to effectively compare them to select the best one. Reproducible Feature Engineering & Model Training: Commit the results of your experiments and use the lakeFS Git integration to reproduce any experiment with the right version of the data, the code and the model weights.

Aber auch im Data Engineering gibt es Vorteile: Isolated Dev/Test Environments: Create isolated dev/test environments using lakeFS branches and reduce your testing time by 80%. Promote Only High Quality Data to Production: Implement CI/CD for data with lakeFS hooks, allowing for automation of quality validation checks. Fix Bad Data with Production Rollback: Save entire consistent snapshots of your data using commits, allowing you to rollback to previous commits in case of bad data.

Den Erfolg und die Effizienz von LakeFS wird durch einige große Kunden wie z.B. Netflix, Daimler und Air Asia, bewiesen.

 

Wir haben LakeFS in einer Docker-Umgebung mit MinIO als lokalen Object Store und Trino als Compute Engine genutzt. Dies war für unseren Anwendungsfall ausreichend.   LakeFS unterstützt jedoch viele Integrationsmöglichkeiten und kann daher in vielen unterschiedlichen Data Stacks verwendet werden. Im Folgenden wird eine kleine Übersicht der derzeitigen Integrationen gegeben. Für eine vollständige Darstellung wird jedoch ausdrücklich auf die Website und Dokumentation von LakeFS verwiesen.

Zu den Integrationsmöglichkeiten gehören:

  • Für den Object Store jeden S3 fähigen Object Store, wie z.B. AWS, Azure, Google Cloud oder auch MinIO.
  • Als Compute Engine die am häufigsten genutzten, wie AWS Glue, Databricks, Snowflake Starburst, Hive, uvm.
  • LakeFS ist unabhängig vom Datenformat. Daher unterstützt es fast alle Datenformate wie z.B. csv oder parquet, aber auch Open Table Formate wie Delta Lake und Iceberg und auch unstrukturierte Daten, wie Bilder und Videos.
  • Zusätzlich unterstützt LakeFS auch Orchestration und Workflow Tools, wie Airflow, Dagster und dbt.
